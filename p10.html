My Calendar I

Implement a MyCalendar class to store your events. A new event can be added if adding the event will not cause a double booking.

Your class will have the method, book(int start, int end). Formally, this represents a booking on the half open interval [start, end), the range of real numbers x such that start <= x < end.

A double booking happens when two events have some non-empty intersection (ie., there is some time that is common to both events.)

For each call to the method MyCalendar.book, return true if the event can be added to the calendar successfully without causing a double booking. Otherwise, return false and do not add the event to the calendar.

Code
                            
                                class MyCalendar:

                                days = []
                                
                                def __init__(self):
                                    self.days = [0]*31
                                    
                            
                                def book(self, start, end):
                                    
                                    if start > end:
                                        return False
                                    
                                    if end > 31:
                                        return False
                                    
                                    # print(len(self.days))
                                    
                                    for x in range(start, end-1):
                                        if self.days[x] == 1:
                                            return False
                                        else:
                                            self.days[x] = 1
                                    
                                    return True
IDEAL

This is a very interesting problem about getting a calendar done. It stats basically by a class that contains a method book which has a start and end date, those days are going to be busy and you cannot book in those thays anymore.

My solution was pretty simple. I created a 31 days list sized in the init method (constructor) then on the book section I checked that the end date cannot be more than the start (will have no sense at all) then I did iterate the list of the days and add a 1 on the index from the start day to the end date-1 (which is in the description) and check that the spaces where available.
